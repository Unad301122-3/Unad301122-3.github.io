Modelos de desarrollo de software

El proceso de desarrollo de software, como ya lo vimos en la lectura anterior, consiste en un conjunto organizado de pasos, guías o acciones que indican qué hacer para obtener un producto o resultado, en este caso un software, que satisfaga las necesidades del cliente o de la empresa para el cual fue creado. Dicho proceso puede definir como iniciar un proyecto, las acciones a seguir y como terminarlo. Para el desarrollo de software han surgido modelos que plantean distintas manera para cumplir con tal propósito, estos modelos son representaciones abstractas de los procesos que se pueden utilizar en el desarrollo de un software. A continuación reseñamos algunos de dichos modelos.

Modelo en cascada.

Presenta una serie de fases o etapas secuenciales y para pasar de una etapa a otra se debe realizar una revisión para verificar que se haya cumplido totalmente una determinada etapa para poder pasar a la siguiente. Las fases son: definición de requerimientos, diseño del software, implementación o construcción del software, integración y prueba del sistema, funcionamiento y mantenimiento.

Modelo en espiral.

En este modelo el software se desarrolla en una serie de entregas llamadas evolutivas, que consiste en que se desarrolla un software inicial para que el cliente lo pruebe, en la segunda entrega se han adicionado más requerimientos, se han corregido errores o se han hecho las modificaciones pertinentes, hasta hacer una entrega final y definitiva del software. Las fases que en el modelo en cascada se realizan por separado, en el modelo en espiral se entrelazan realizándose una retroalimentación entre estas.

De acuerdo a los análisis de la industria del software, este modelo es más aplicable a sistemas pequeños, no viable para el desarrollo de grandes softwares.

Modelo basado en componentes.

Este modelo se basa en la reutilización de componentes de software existente que pueden ser reutilizados en la creación de un nuevo software. Este modelo reduce la cantidad de software que se debe desarrollar y reduce los costos y los riesgos. La reutilización de códigos de software también puede ser empleada en los otros modelos.

En cuanto a la calidad del software, también han surgido normas y modelos que pretenden brindar orientaciones para mejorar en los procesos de producción de software que podemos ver en la siguiente lectura.
